How the proof of concept works

The notebook already provides the inference path. The website just wraps it in a user-friendly flow.

1
Collect sequences

User pastes a peptide and protein sequence, or selects a record by UniProt or PDB ID.

2
Generate embeddings

The backend converts both sequences into ProtT5-style vectors.

3
Run the model

The 2048-feature vector is passed to the trained Keras model.

4
Show ranked hits

The UI displays probability, threshold status, and export options.
